Angular Host

Container that integrates and renders components from other frameworks

Responsibilities:

  • Configure remotes in webpack
  • Dynamically load remote components
  • Handle cross-framework communication

React Remote

Exposes React components for use in the Angular host application

Exposed Components:

  • UI components (buttons, cards)
  • Feature modules
  • React hooks & utilities

Vue.js Remote

Exposes Vue components for use in the Angular host application

Exposed Components:

  • Advanced UI widgets
  • Specialized forms
  • Data visualization tools

How Module Federation Works

Simple overview of Module Federation with Angular as host

Angular Host

Loads and renders remote components

remotes: {{ '{' }}
react_app: '...',
vue_app: '...'
{{ '}' }}

React Remote

Exposes React components

exposes: {{ '{' }}
'./Button': '...',
'./Card': '...'
{{ '}' }}

Vue Remote

Exposes Vue components

exposes: {{ '{' }}
'./DatePicker': '...',
'./Modal': '...'
{{ '}' }}
1

Configuration

Each application configures Module Federation in its webpack config. The host defines remote entry points, and remote apps expose their components.

2

Remote Entry

Remote applications generate a remoteEntry.js file that acts as a manifest, telling the host which modules are available and how to load them.

3

Dynamic Loading

The Angular host dynamically imports remote components at runtime using special import syntax that references the remote application.

4

Framework Bridging

Angular wraps remote components as custom elements or creates framework-specific adapters to render React and Vue components in the Angular context.

5

Shared Dependencies

Common libraries are shared between applications to avoid duplication and ensure consistent behavior. Each app declares which dependencies can be shared.

Key Benefits

Independent Teams

Teams can work independently using their preferred frameworks while still contributing to a unified application.

Improved Loading

Components load on demand, improving initial load time and enabling progressive enhancement.

Framework Agnostic

Choose the best technology for each component without committing the entire application to one framework.
